# Contributing to Agentic Prompts

Thank you for considering a contribution.

This repository is focused on practical, production-grade prompts for engineers using AI agents and AI-assisted development tools.

---

## What You Can Contribute

You can contribute:

- New prompts
- Improved prompt versions
- Prompt examples
- Tool-specific adaptations
- Cursor rules
- Claude Code workflows
- Codex task templates
- Real-world usage examples

---

## Prompt Quality Checklist

Before submitting a prompt, check that it includes:

- [ ] Clear role definition
- [ ] Clear objective
- [ ] Scope boundaries
- [ ] Step-by-step workflow
- [ ] Expected output format
- [ ] Safety rules
- [ ] Validation requirements
- [ ] Constraints
- [ ] Final reporting format

---

## Recommended Prompt File Format

```markdown
# Prompt Title

## Purpose

Explain what this prompt is used for.

## Best Used With

- ChatGPT
- Claude
- Cursor
- Claude Code
- Codex

## Prompt

```text
Your prompt goes here.
```

## Notes

Add usage notes, limitations, or examples.
```

---

## Contribution Rules

Please make sure your prompt is:

- Practical
- Clear
- Safe
- Structured
- Production-oriented
- Reusable

Avoid prompts that:

- Encourage unsafe code changes
- Ask agents to guess without evidence
- Promote hardcoded secrets
- Skip validation
- Encourage unnecessary rewrites
- Are too vague to be useful

---

## Pull Request Guidelines

When submitting a pull request:

1. Add the prompt file in the correct folder.
2. Update `README.md` with the new prompt link.
3. Keep the description short and clear.
4. Explain when the prompt should be used.
5. Mention which tools it works best with.

---

## Tone and Style

Prompts should sound like professional engineering instructions.

Prefer:

- Clear
- Direct
- Structured
- Actionable

Avoid:

- Overly motivational language
- Unclear wording
- Long paragraphs without structure
- Tool-specific claims that are not tested
