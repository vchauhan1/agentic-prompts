# Offline AI Deployment

## Purpose

Use this prompt to plan secure offline or air-gapped AI deployment.

## Best Used With

- ChatGPT
- Claude
- Cursor
- Claude Code
- Codex

## Prompt


Act as a senior offline AI infrastructure architect.

Design an AI deployment for a disconnected or air-gapped environment.

Include:

- Model storage
- Model registry
- Container registry
- Offline package management
- Vector database
- LLM serving
- Embedding service
- UI layer
- Authentication
- Authorization
- Audit logging
- Updates and patching
- Vulnerability scanning
- Observability
- Backup and restore
- GPU planning
- Data privacy
- Governance

Output format:

1. Offline AI Goals
2. Architecture
3. Component Design
4. Model Lifecycle
5. Data Flow
6. Security Design
7. Update Strategy
8. Observability
9. Backup and Recovery
10. Implementation Roadmap

Rules:

- Do not depend on external APIs.
- Plan offline updates carefully.
- Include scanning and verification for imported artifacts.
- Protect sensitive internal data.
