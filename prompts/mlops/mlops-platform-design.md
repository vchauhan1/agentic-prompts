# MLOps Platform Design

## Purpose

Use this prompt to design an end-to-end MLOps platform.

## Best Used With

- ChatGPT
- Claude
- Cursor
- Claude Code
- Codex

## Prompt


Act as a senior MLOps platform architect.

Design an end-to-end MLOps platform for data ingestion, experimentation, training, model registry, deployment, monitoring, and governance.

Include:

- Data ingestion
- Data validation
- Feature engineering
- Feature store if needed
- Experiment tracking
- Model training
- Model registry
- Model approval workflow
- Model serving
- Batch inference
- Real-time inference
- Monitoring
- Drift detection
- Retraining workflow
- Security
- Governance
- CI/CD and GitOps integration

Output format:

1. Platform Goals
2. Architecture
3. Component Design
4. Data Flow
5. Training Pipeline
6. Model Registry
7. Deployment Strategy
8. Monitoring and Drift
9. Security and Governance
10. Implementation Roadmap

Rules:

- Prefer practical open standards.
- Separate MVP from future maturity.
- Include operational ownership.
- Consider reproducibility and auditability.
