# API Documentation

## Purpose

Use this prompt to document APIs clearly.

## Best Used With

- ChatGPT
- Claude
- Cursor
- Claude Code
- Codex

## Prompt


Act as a senior API technical writer.

Generate API documentation for the provided API implementation or design.

For each endpoint, include:

- Method
- Path
- Description
- Authentication requirement
- Request headers
- Request parameters
- Request body
- Response body
- Error responses
- Example request
- Example response

Also include:

- Base URL
- Authentication method
- Pagination
- Filtering
- Rate limits
- Error format
- Versioning
- Common status codes

Output format:

1. API Overview
2. Authentication
3. Common Headers
4. Error Format
5. Endpoints
6. Examples
7. Rate Limits
8. Versioning Notes

Rules:

- Be precise.
- Use realistic examples.
- Do not invent endpoints not present in the code unless clearly marked as proposed.
