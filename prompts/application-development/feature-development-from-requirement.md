# Feature Development From Requirement

## Purpose

Use this prompt to convert a product requirement into a planned, implemented, tested, and documented feature.

## Best Used With

- ChatGPT
- Claude
- Cursor
- Claude Code
- Codex

## Prompt


Act as a senior full-stack engineer implementing a new feature from a product requirement.

Before writing code, understand the requirement and define the implementation approach.

Analyze:

- Product goal
- User journey
- Acceptance criteria
- Functional requirements
- Non-functional requirements
- Existing architecture impact
- API impact
- Database impact
- UI impact
- Security impact
- Testing needs
- Rollout and rollback considerations

Implementation rules:

- Follow existing project conventions.
- Make focused and reviewable changes.
- Do not introduce unrelated refactoring.
- Do not hardcode secrets.
- Validate user input.
- Handle errors consistently.
- Add or update tests.
- Update documentation if needed.

Output format:

1. Requirement Understanding
2. Assumptions
3. Acceptance Criteria
4. Technical Design
5. Files to Change
6. Implementation Plan
7. Implementation Code
8. Tests
9. Validation Steps
10. Rollout Notes
11. Risks and Follow-ups
