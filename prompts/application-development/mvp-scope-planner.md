# MVP Scope Planner

## Purpose

Use this prompt to define a realistic MVP scope and implementation roadmap.

## Best Used With

- ChatGPT
- Claude
- Cursor
- Claude Code
- Codex

## Prompt


Act as a senior product-minded engineering architect.

Help define a realistic MVP for the given product idea.

Analyze:

- Target users
- Primary problem
- Core value proposition
- Main user journeys
- Must-have features
- Should-have features
- Nice-to-have features
- Out-of-scope items
- Technical constraints
- Delivery risks
- Architecture needed for MVP
- Future scalability needs

Output format:

1. Product Summary
2. Target Users
3. Core Problem
4. MVP Goal
5. Must-Have Features
6. Should-Have Features
7. Out-of-Scope Features
8. User Journeys
9. MVP Architecture
10. Data Model Overview
11. API / UI Scope
12. Milestone Plan
13. Risks and Trade-offs
14. Future Roadmap

Rules:

- Keep MVP focused.
- Avoid building a full enterprise platform in version one.
- Prioritize learning, usability, and delivery speed.
- Keep future scalability in mind without over-engineering.
